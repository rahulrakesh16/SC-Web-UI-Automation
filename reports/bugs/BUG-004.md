## BUG-004: Division Operands Are Reversed — a ÷ b Computes b / a

| Field             | Detail                                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| **Severity**      | High                                                                                                       |
| **Priority**      | P1                                                                                                         |
| **Status**        | Open                                                                                                       |
| **Type**          | Functional Defect — Incorrect Arithmetic Logic                                                             |
| **Environment**   | Chromium / Firefox / WebKit — macOS / Ubuntu / https://rbihubcodechallenge.github.io/calculator/index.html |
| **Discovered By** | Automated regression suite — TC-BUG-004                                                                    |

### Preconditions

- Calculator page is loaded and display is empty.

### Steps to Reproduce

1. Navigate to `https://rbihubcodechallenge.github.io/calculator/index.html`
2. Click **6**, then **÷**, then **2**, then **=**
3. Observe the displayed result

### Expected Result

`6 ÷ 2 = 3`

### Actual Result

`6 ÷ 2 = 0.3333333333333333` (computes `2 ÷ 6` instead)

**Root Cause (obfuscated JS analysis — recursive descent parser):**

In the multiplication/division handler `_0x8329bc()`:

```javascript
let firstOperand = parseNextToken(); // reads '6'
// operator '/' consumed
let secondOperand = parseNextToken(); // reads '2'
result = secondOperand / firstOperand; // BUG: 2/6 instead of 6/2
```

The dividend and divisor are swapped — `secondOperand / firstOperand` instead of
`firstOperand / secondOperand`. Multiplication is unaffected because `a * b === b * a`.

**Side-effect of combined BUG-002 + BUG-004:**

- Pressing `0 ÷ 5` computes `5 / 0 = Infinity` (zero becomes denominator due to swap)
- Pressing `5 ÷ 0` computes `0 / 5 = 0` (fails silently — no division-by-zero error)

### Banking Domain Impact

Division is the most common arithmetic operation in finance: interest rate calculations,
yield computations, share price per earnings, currency conversion, fee splits.
A reversed division produces results that are the **reciprocal** of the correct answer.

Example: A banker computing a loan repayment of £120,000 over 12 months:

- Expected monthly instalment: `120000 ÷ 12 = £10,000`
- Actual result from calculator: `12 ÷ 120000 = £0.0001`

This is a **critical financial accuracy defect**. The severity is High (not Critical) only because
sin() is typically less frequently used in core banking workflows than division.
