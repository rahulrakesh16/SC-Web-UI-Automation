## BUG-001: Subtraction Button Appends Division Operator Instead of Minus Sign

| Field             | Detail                                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| **Severity**      | Critical                                                                                                   |
| **Priority**      | P1                                                                                                         |
| **Status**        | Open                                                                                                       |
| **Type**          | Functional Defect — Wrong Operator Mapping                                                                 |
| **Environment**   | Chromium / Firefox / WebKit — macOS / Ubuntu / https://rbihubcodechallenge.github.io/calculator/index.html |
| **Discovered By** | Automated regression suite — TC-BUG-001                                                                    |

### Preconditions

- Calculator page is loaded and display is empty.

### Steps to Reproduce

1. Navigate to `https://rbihubcodechallenge.github.io/calculator/index.html`
2. Click the button labeled **"−"** (visually a minus/subtraction sign)
3. Observe the value appended to the display

### Expected Result

Display shows **`-`** — the subtraction operator is appended, enabling expressions such as `9 - 4 = 5`.

### Actual Result

Display shows **`/`** — the division operator is appended instead.

**Root Cause (DOM inspection):**

```html
<button onclick="append('/')">−</button>
```

The button label reads "−" but the `onclick` handler calls `append('/')`, appending a forward slash.
There is **no button on the entire calculator that appends the subtraction operator**. Subtraction
functionality is completely absent from this application.

### Banking Domain Impact

A financial calculator without subtraction is critically defective. Any computation involving debt
repayment, balance reduction, profit/loss calculation, or interest deduction will produce incorrect
results or be impossible to perform. This is a **showstopper defect** for any banking use case.

### Evidence

- Automated test: `tests/regression/bugs.spec.ts` → `TC-BUG-001` (FAILS)
- DOM source: `onclick="append('/')"` on button element with text content "−"
- Playwright MCP DOM inspection confirms `normalize-space()='−'` maps to `append('/')`
