## BUG-003: sin() Function Always Returns 1 Regardless of Input

| Field             | Detail                                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| **Severity**      | High                                                                                                       |
| **Priority**      | P1                                                                                                         |
| **Status**        | Open                                                                                                       |
| **Type**          | Functional Defect — Hardcoded Return Value                                                                 |
| **Environment**   | Chromium / Firefox / WebKit — macOS / Ubuntu / https://rbihubcodechallenge.github.io/calculator/index.html |
| **Discovered By** | Automated regression suite — TC-BUG-003                                                                    |

### Preconditions

- Calculator page is loaded and display is empty.

### Steps to Reproduce

1. Navigate to `https://rbihubcodechallenge.github.io/calculator/index.html`
2. Enter any valid numeric value (e.g., `0`, `90`, `45`)
3. Click the **"sin"** button
4. Observe the displayed result

### Expected Result

| Input | Expected sin() Result     |
| ----- | ------------------------- |
| 0     | 0                         |
| 90    | 0.8939966636... (radians) |
| 45    | 0.8509035245... (radians) |

### Actual Result

**Always returns `1`**, regardless of the input value.

**Root Cause (obfuscated JS analysis):**

```javascript
if (type === 'sin') display.value = 434563 ^ 434562;
// 434563 XOR 434562 = 1 (bitwise XOR of adjacent integers always yields 1)
```

The `sin` branch in the `func()` function assigns a hardcoded integer XOR expression
`434563^434562` (which evaluates to `1`) instead of calling `Math.sin()`. The obfuscation
obscures this as unicode-escaped characters, but the evaluated result is always `1`.

In contrast, `cos`, `tan`, `sqrt`, and `log` all correctly call their respective `Math.*` methods.

### Banking Domain Impact

Trigonometric functions are used in bond pricing, option pricing (Black-Scholes), and
actuarial calculations. A `sin()` that always returns `1` would silently corrupt any
financial model relying on this function, producing wrong results without any error indication.
The silent failure mode (no `Error` displayed) is particularly dangerous.

- Automated test: `tests/regression/bugs.spec.ts` → `TC-BUG-003` (FAILS)
- Also documented in: `tests/functional/scientific.spec.ts` → `TC-SCI-009`, `TC-SCI-010`
