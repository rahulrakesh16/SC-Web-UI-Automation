## BUG-008: Double Decimal Point Input Is Silently Accepted and Produces Incorrect Result

| Field             | Detail                                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| **Severity**      | Low                                                                                                        |
| **Priority**      | P3                                                                                                         |
| **Status**        | Open                                                                                                       |
| **Type**          | Functional Defect — Missing Input Validation                                                               |
| **Environment**   | Chromium / Firefox / WebKit — macOS / Ubuntu / https://rbihubcodechallenge.github.io/calculator/index.html |
| **Discovered By** | Automated test failure — TC-BOUND-011 / exploratory testing                                                |

### Preconditions

- Calculator page is loaded and display is empty.

### Steps to Reproduce

1. Navigate to `https://rbihubcodechallenge.github.io/calculator/index.html`
2. Click **1**
3. Click **.** (decimal point)
4. Click **.** (decimal point again)
5. Click **5**
6. Click **=**

Display after step 5 reads: `1..5`

### Expected Result

One of the following should occur:

- Display shows **`NaN`** — evaluating `1..5` is syntactically invalid
- Display shows **`Error`** — calculator rejects the malformed input
- Second **.** is ignored / prevented — display remains `1.`

### Actual Result

Display shows **`1`** — the parser silently drops the second decimal point **and the digit `5`
that follows it**, evaluating only the integer portion `1`.

**Root Cause (Code Analysis):**
The tokeniser splits the display string on operator boundaries. When it encounters `1..5`:

1. It splits on the first `.` producing the token `1.` (incomplete float)
2. It then sees `.5` as a second token starting with `.`
3. The second `.` is not treated as an error; it starts a new float literal `.5` = `0.5`
4. However, the second token is apparently not incorporated into the expression, so only `1` (or
   `1.` → `1`) is evaluated

The net effect: `1..5` silently evaluates as `1`. No error, no warning.

### Banking Domain Impact

While rated Low severity due to limited blast radius (it affects only manual decimal entry of
individual numbers), the silent acceptance of malformed input is a symptom of insufficient
input validation across the board:

- A user attempting to type `1.5` who accidentally double-taps `.` gets `1` silently
- In financial contexts, `1.5%` becoming `1%` introduces a 50% relative error in rate calculations
- The lack of any error message violates the principle of least surprise and can cause untraceable
  data entry mistakes

### Evidence

- **Automated test (FAIL):**
  - `src/tests/regression/bugs.spec.ts` → `TC-BUG-008` (FAILS — `"1..5"` returns `"1"` not NaN/Error)
  - `src/tests/boundary/boundary.spec.ts` → TC-BOUND-011 (documents actual `"1"` behavior)
- **Exploratory proof (JS console):**
  ```javascript
  clearDisplay();
  append('1');
  append('.');
  append('.');
  append('5');
  calculate();
  document.getElementById('display').value; // → "1" (expected NaN or Error)
  ```

### Suggested Fix

Guard against duplicate decimal points in the `append()` function:

```javascript
function append(value) {
  const display = document.getElementById('display');
  const current = display.value;
  // Prevent second decimal in the current number token
  if (value === '.') {
    const lastOperatorIndex = Math.max(
      current.lastIndexOf('+'),
      current.lastIndexOf('*'),
      current.lastIndexOf('/'),
      current.lastIndexOf('('),
    );
    const currentToken = current.slice(lastOperatorIndex + 1);
    if (currentToken.includes('.')) return; // Ignore duplicate decimal
  }
  display.value += value;
}
```
