## BUG-002: Button Labeled "3" Appends Digit "0" Instead of "3"

| Field             | Detail                                                                                                     |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| **Severity**      | Critical                                                                                                   |
| **Priority**      | P1                                                                                                         |
| **Status**        | Open                                                                                                       |
| **Type**          | Functional Defect — Wrong Digit Mapping                                                                    |
| **Environment**   | Chromium / Firefox / WebKit — macOS / Ubuntu / https://rbihubcodechallenge.github.io/calculator/index.html |
| **Discovered By** | Automated regression suite — TC-BUG-002                                                                    |

### Preconditions

- Calculator page is loaded and display is empty.

### Steps to Reproduce

1. Navigate to `https://rbihubcodechallenge.github.io/calculator/index.html`
2. Click the button visually labeled **"3"** in the number grid
3. Observe the value appended to the display

### Expected Result

Display shows **`3`** — the digit three is entered.

### Actual Result

Display shows **`0`** — the digit zero is appended instead.

**Root Cause (DOM inspection):**

```html
<button onclick="append('0')">3</button>
```

The button label reads "3" but the `onclick` handler calls `append('0')`. Digit **3 is completely
inaccessible**. Additionally, the real `0` button at position 17 also appends `'0'`, meaning there
are two buttons appending `0` and none appending `3`.

### Banking Domain Impact

Any financial amount or calculation involving the digit 3 is impossible to enter correctly.
Values such as £13.50, £30,000, £333.33 (common loan amounts, tax rates, interest amounts)
cannot be input. This defect silently corrupts numeric input with no user-visible error,
which is particularly dangerous in a financial context.

### Evidence

- Automated test: `tests/regression/bugs.spec.ts` → `TC-BUG-002` (FAILS)
- DOM source: `onclick="append('0')"` on button element with text content "3"
- Also documented in: `tests/functional/arithmetic.spec.ts` → `TC-ARITH-013`
